## 문제 내용

이 문제에서 정규표현식의 정의는 다음과 같습니다.

* `a`와 `b`는 각각 정확히 `"a"`, `"b"`와만 매치되는 정규표현식입니다.
* `R1`과 `R2`가 정규표현식일 때, `(R1R2)`는 `R1`에 매치되는 모든 문자열 `s1`과 `R2`에 매치되는 모든 문자열 `s2`에 대해 `s1s2` (두 문자열을 연결한 것)와 매치되는 정규표현식입니다.
* `R1`과 `R2`가 정규표현식일 때, `(R1|R2)`는 `R1`에 매치되거나 `R2`에 매치되는 문자열과 매치되는 정규표현식입니다.
* `R1`이 정규표현식일 때, `(R1*)`는 `R1`에 매치되는 0개 이상의 문자열을 연결한 것과 매치되는 정규표현식입니다.

두 문자열 `s1`과 `s2`의 편집 거리는 `s1`을 `s2`로 바꾸기 위해 필요한 삽입, 삭제, 또는 치환의 최소 횟수입니다.

* 삽입: 원하는 한 글자를 임의의 위치에 삽입
* 삭제: 임의의 위치의 한 글자를 삭제
* 치환: 임의의 위치의 한 글자를 다른 글자로 치환

정규표현식 `R1`과 `R2`가 주어질 때, `R1`에 매치되는 문자열과 `R2`에 매치되는 문자열 사이의 편집 거리의 최솟값을 구하세요.

### 입력

첫 번째 줄에는 테스트 케이스의 개수 $T$가 주어집니다. ($1 \le T \le 85$)

각 테스트 케이스에 대해, 정규표현식 `R1`과 `R2`가 각각 한 줄에 주어집니다. ($1 \le |R_1|, |R_2| \le 50$) 두 테스트 케이스 사이에는 빈 줄이 주어집니다.

### 출력

각 테스트 케이스에 대해, 문제의 정답을 한 줄에 출력합니다.

## 문제 풀이

{{< details title="스포일러" closed="true" >}}

먼저, 두 개의 정규표현식을 ε-NFA로 변환합니다. 다음과 같은 방법을 사용하면 start state와 accept state가 각각 1개인 ε-NFA를 만들 수 있으며, state의 개수는 문자열 길이 + O(1),
transition의 개수는 문자열 길이의 2배 + O(1)로 만들 수 있습니다.

* `a`와 `b`는 각각 state 2개짜리 NFA로 만듭니다.
* `R1R2`는 `R1`의 accept state에서 `R2`의 start state로 ε-transition을 삽입합니다.
* `R1|R2`는 start state와 accept state를 별도로 만들고, 새로운 start state에서 `R1`과 `R2`의 start state로, `R1`과 `R2`의 accept state에서 새로운 accept state로 ε-transition을 삽입합니다.
* `R1*`는 기존 accept state에서 start state로 ε-transition을 삽입하고, 새로운 accept state를 만들어 start state로부터 들어오는 ε-transition을 삽입합니다.

이제 문제의 답을 구하기 위해서는 편집 거리 DP의 전이를 응용할 수 있습니다.

* 둘 중 한쪽 NFA에서 ε-transition을 통해 이동하는 것은 비용이 0입니다.
* 한쪽 NFA에서 a 또는 b transition을 통해 이동하는 것은 삽입이나 삭제에 대응되며, 따라서 비용이 1입니다.
* 양쪽 NFA에서 같은 글자에 해당하는 transition을 통해 이동하는 것은 비용이 0입니다.
* 양쪽 NFA에서 다른 글자에 해당하는 transition을 통해 이동하는 것은 치환에 대응되며, 비용이 1입니다.

이렇게 정의하고 (start1, start2)에서 시작하여 0-1 BFS를 돌린 뒤에, (accept1, accept2)의 거리를 출력하면 됩니다.
