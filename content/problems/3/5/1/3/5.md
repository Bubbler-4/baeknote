## 문제 내용

Xordition Robot은 다음과 같은 두 종류의 카드를 끼워넣고 정수 하나를 입력하면 카드 순서대로 연산을 하여 결과를 알려줍니다.

* `+ i` ($0 \le i < 16$): 현재 수에 $i$를 더하고 모듈로 16을 취합니다.
* `x i` ($0 \le i < 16$): 현재 수에 $i$를 XOR합니다.

최초에 로봇에 끼워져 있는 $N$장의 카드가 주어집니다. 다음의 쿼리를 $Q$개 처리하세요.

* `i c j` ($1 \le i \le N$, $c$는 `+`와 `x` 중 하나, $0 \le j < 16$): 앞에서 $i$번째 카드를 `c j`로 교체한 뒤, 로봇에게 0을 입력하였을 때 출력값을 출력합니다.

### 입력

첫 번째 줄에 $N$과 $Q$가 주어집니다. ($1 \le N \le 200\;000$, $1 \le Q \le 200\;000$)

다음 $N$줄에는 처음에 끼워져 있는 카드들이 한 줄에 하나씩 순서대로 주어집니다.

다음 $Q$줄에는 쿼리가 한 줄에 하나씩 주어집니다.

### 출력

각 쿼리에 대한 정답을 한 줄에 하나씩 출력합니다.

## 문제 풀이

{{< details title="스포일러" closed="true" >}}

이 문제는 세그먼트 트리로 풀 수 있습니다. 덧셈과 XOR는 그 자체로는 합성이 잘 되지 않지만, 각각의 카드는 정의역과 공역이 각각 $\{0, 1, \cdots, 15\}$인 함수이므로, 16개 입력에 대한 결과값들을 유지하면
함수 합성을 세그먼트 트리 연산으로 볼 수 있습니다.

각 쿼리에 대해, $i$번째 카드를 업데이트하고, 트리의 전체 구간을 쿼리한 결과의 0번째 원소를 출력하면 됩니다.
