## 문제 풀이

{{< details title="스포일러" closed="true" >}}

시간 복잡도 표현을 위해서, 페이지 번호와 캐시 크기의 최댓값을 $A$라고 합시다. 문제에서 $A = 20$입니다.

각각의 페이지 번호 $x$에 대해서, 페이지 부재는 다음과 같은 상황에 발생할 수 있습니다.

* type 1: $x$번 페이지를 처음으로 불러올 때
* type 2: $x$번 페이지를 불러온 적 있으나 다음으로 $x$번 페이지를 불러오기 전에 날아갔을 때
  * 이 경우는 두 번의 불러오기 사이에 $k$종류 이상의 페이지를 불러왔을 때 발생하며, $k$종류 미만의 페이지를 불러왔을 때는 발생하지 않습니다.

이 두 가지에 대한 구간 쿼리를 하기 위해서, 다음과 같은 세그먼트 트리 노드를 구상합니다.

```none
struct Node {
    int whole,      // 전체 구간에 등장하는 페이지 집합 (비트마스크)
    int left[20],   // left[x]: 첫 번째 x번 페이지의 왼쪽에 등장하는 페이지 집합
    int right[20],  // right[x]: 마지막 x번 페이지의 오른쪽에 등장하는 페이지 집합
    int evicts[20], // evicts2[i]를 k = i+1일 때 type 2 페이지 부재의 횟수라고 할 때,
                    // evicts는 evicts2의 차분 배열
};
```

두 구간을 합할 때, 각각의 페이지 번호 $x$에 대해서 왼쪽 구간의 마지막 $x$와 오른쪽 구간의 첫 번째 $x$ 사이에서 type 2 페이지 부재가 발생할 수 있습니다.
따라서 그 두 집합의 합집합의 크기 $s$에 따라서 `evicts2`의 값을 증가시켜야 하는데, 증가되는 값은 $k \le s$에서 +1입니다.
이를 나이브하게 처리하면 각 페이지에 대해 최대 $A$개의 업데이트를 해야 하므로 총 $A^2$의 시간이 걸리게 됩니다. 이를 $A$로 줄이기 위해서 차분 배열을 도입할 수 있습니다.

한 구간에 페이지 $x$가 아예 존재하지 않을 경우, `left[x]`와 `right[x]`는 모두 `-1`로 두고, 반대편 구간에 $x$가 존재하는지 여부를 이용하여 합친 결과의 `left[x]`와 `right[x]`를 적절히 업데이트할 수 있습니다.

구간 쿼리의 답을 구하려면 type 1과 type 2 페이지 부재의 횟수를 더해야 합니다. type 1 페이지 부재는 `whole`의 켜진 비트의 개수로 구할 수 있고, type 2 페이지 부재는
`evicts`의 첫 $k$개 값의 합으로 구할 수 있습니다.
